# Contributing to O-Walkie

Thanks for your interest in contributing.

## Ways to Contribute

- Open issues for bugs, regressions, UX problems, or documentation gaps.
- Suggest features and protocol improvements.
- Submit pull requests with focused, reviewable changes.

## Before You Start

1. Check existing issues/PRs to avoid duplicates.
2. For larger work, open an issue first and align on scope.
3. Keep backward compatibility in mind for relay protocol changes.

## Pull Request Guidelines

- Keep PRs small and focused.
- Include clear rationale ("why"), not only implementation details ("what").
- Add/update docs when behavior or configuration changes.
- Test affected components locally (backend, Android, Windows prototype when relevant).

## Commit Message Guidance

Use concise imperative style, for example:
- `Fix Android profile navigation with empty lists.`
- `Add configurable relay filter module.`

## Code Style

- Follow existing style per component/language.
- Prefer readability over cleverness.
- Avoid unrelated refactoring in functional PRs.

## Security

- Never commit secrets, keys, tokens, or private credentials.
- Use sanitized examples in config and docs.
